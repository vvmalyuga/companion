Фреймовое моделирование

Фрейм-модели — метод структурного представления знаний о предметной области через фреймы (кадры).

Структурированность

Четкая организация знаний

Иерархичность

Возможность наследования свойств

Модульность

Независимое описание компонентов

Типы фрейм-моделей

Функция по предметной области

  • Поиск компаньонов
  • Бронирование встреч
  • Обработка платежей

Внешняя сущность

  • Пользователь
  • Платежная система
  • СМС-сервис

Хранилище данных

  • База пользователей
  • Таблица заказов
  • Архив отзывов

Поток данных

  • Запрос на поиск
  • Ответ с результатами
  • Платежная транзакция

Роль

  • Заказчик
  • Компаньон
  • Администратор

Прецедент

  • Регистрация пользователя
  • Поиск компаньона
  • Оформление бронирования

Примеры фрейм-моделей

Фрейм "Content" (Контент)

Структура:

Фрейм: Content
Слоты:
  - id: Уникальный идентификатор
  - type: Тип контента
  - owner: Владелец
  - status: Статус
  - visibility: Видимость

Фрейм "Role" (Роль)

Структура:

Фрейм: Role
Слоты:
  - role_id: Идентификатор
  - role_name: Название
  - permissions: Разрешения
  - hierarchy_level: Уровень

Фрейм "Class" (Класс)

Структура:

Фрейм: Class
Слоты:
  - class_name: Имя класса
  - attributes: Атрибуты
  - methods: Методы
  - parent_class: Родительский класс

Дополнительные фреймы

Фрейм "BusinessProcess"

Фрейм: BusinessProcess
Слоты:
  - process_id: Идентификатор
  - process_name: Название
  - steps: Шаги выполнения
  - actors: Участники
  - inputs: Входные данные
  - outputs: Выходные данные

Фрейм "Data"

Фрейм: Data
Слоты:
  - data_id: Идентификатор
  - data_type: Тип данных
  - format: Формат
  - size: Размер
  - sensitivity: Чувствительность

Фрейм "Interaction"

Фрейм: Interaction
Слоты:
  - interaction_id: Идентификатор
  - from: Отправитель
  - to: Получатель
  - message_type: Тип сообщения
  - content: Содержание
